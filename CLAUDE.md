## Project

Unity 6 (6000.4.9f1) test-task project. Render pipeline: URP. Input: new Input System. Single scene at `Assets/Scenes/SampleScene.unity`. No README, no Cursor/Copilot rules.

The repo is small and intentionally minimal — the first commit message describes it as "a simple reactive system" (in Russian). Most of `Assets/Scripts/` is the reactive primitives plus an MVVM-style View base; ViewModels are not yet written (`Assets/Scripts/GUI/ViewModels/` is empty).

## Build / run / test

There is no CLI build script — open the project in Unity 6000.4.9f1 (Editor) and use the Editor to enter Play mode, build, and run EditMode/PlayMode tests via **Window → General → Test Runner**. The `com.unity.test-framework` package is installed but no test assemblies exist yet.

The `.sln` / `.csproj` files are Unity-generated; do not hand-edit them — they are regenerated by the Editor and by Rider/VS on import.

## External dependencies worth knowing

Pulled from `Packages/manifest.json`:

- **UniTask** (`com.cysharp.unitask`, git package) — prefer `UniTask` over `Task`/coroutines for async work.
- **VContainer** (`jp.hadashikick.vcontainer`, git package, 1.18.0) — DI container. No `LifetimeScope` is wired up yet; if you add one, it is the expected place to compose ViewModels and inject `IReactiveProperty<T>` into Views.

## Architecture

### Reactive layer (`Assets/Scripts/Reactive/`)

A hand-rolled mini-Rx, deliberately narrow:

- `IObserved<T>` — read side: `Subscribe(Action<T>, bool invokeImmediately = true)`, `Unsubscribe`, `OnChanged` event.
- `IReactiveProperty<T> : IObserved<T>` — adds `Value { get; set; }`.
- `ReactiveProperty<T>` — the implementation. Setter is **equality-gated** via `EqualityComparer<T>.Default` (no-op when value is unchanged); use `SetValueAndForceNotify` when you need to push even on equal values. Subscribers are stored in a `List<Action<T>>` and iterated directly — do **not** subscribe/unsubscribe from inside a callback, that will mutate the list during iteration. `Subscribe` returns a `Subscription<T>` (an `IDisposable`); also implements `IDisposable` itself (clears subscribers, sets a disposed flag, suppresses finalize). Note: the public `OnChanged` event is declared but never raised — only the `_subscribers` list is invoked.
- `Subscription<T>` — disposable handle returned by `Subscribe`. Disposing unsubscribes the callback and nulls owner/callback (idempotent).
- `CompositeDisposable` + `DisposableExtensions.AddTo` — standard Rx-style bag for batching disposal in a View's `Release()`.

When extending: keep the surface narrow. Don't pull in UniRx/R3 — the whole point of this folder is that it's the project's own minimal reactive primitives.

### MVVM View layer (`Assets/Scripts/GUI/Views/`)

- `IView<in TViewModel>` — `Bind(TViewModel)` / `Release()`.
- `ViewBase : MonoBehaviour` — empty marker base.
- `View<TViewModel> : ViewBase, IView<TViewModel>` — stores the bound VM, exposes `GetViewModel` (throws if unbound), auto-calls `Bind(_viewModel)` on `OnEnable` and `Release()` on `OnDisable`. Concrete views should override `Bind` to wire reactive subscriptions (typically into a `CompositeDisposable`) and override `Release` to dispose them.

**Known state of this folder (as of writing):** `View.cs` and `ViewBase.cs` were just moved out of `Views/Interface/` up to `Views/`, but their `namespace` is still `GUI.Views.Interface` — only `IView.cs` actually lives in `Interface/`. Decide whether to (a) move the files back, or (b) rename the namespace to `GUI.Views`, before adding new code that references them — don't paper over it by inheriting the mismatch.

### Folder → namespace convention

`Assets/Scripts/<Area>/...` maps to namespace `<Area>` (`Reactive/` → `Reactive`, `GUI/Views/Interface/` → `GUI.Views.Interface`). Keep new files consistent with this so Unity's asmdef-less default compilation stays clean.
